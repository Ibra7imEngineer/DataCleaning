# مشروع تنظيف البيانات (Data Cleaning)

هذا المستودع هو تطبيق وأدوات لمعالجة وتنظيف البيانات النصية والرقمية بهدف تجهيزها للتحليل أو النمذجة. الملف التالي يشرح بنية المشروع، كيفية تشغيل التطبيق واختبارات الوحدة، ووصفًا تفصيليًا للملفات والكود.

## المحتويات الرئيسية

- **الوصف:** مشروع لتنظيف وتحويل البيانات، يتضمن محركات متقدمة لمعالجة القيم المفقودة، تطبيع التواريخ، ومعالجة النصوص.
- **الملفات الأساسية:** `app.py`, `requirements.txt`, مجلد `core/`, مجلد `legacy/`, مجلد `tests/`, ومجلد `config/`.

## المتطلبات

- Python 3.10 أو أعلى (موصى به).
- ينصح بعزل بيئة افتراضية: `python -m venv .venv` ثم تفعيلها.

لتثبيت الاعتمادات:

```
python -m pip install -r requirements.txt
```

````

# README — واجهة مشروع تنظيف البيانات

ملف هذا `README` مصمم ليقدّم وصفًا احترافيًا وشاملاً للموقع/التطبيق والرمز المصدري داخل هذا المستودع. يغطي المستند: الهدف، الميزات، كيفية الإعداد والتشغيل، بنية المشروع، أمثلة استخدام عملية، إجراءات الاختبار، إرشادات النشر، واستكشاف الأخطاء.

## 1. لمحة عامة

هذا المشروع مخصّص لتنظيف وتحضير مجموعات البيانات (نصية ورقمية) لغايات التحليل أو النمذجة. يتضمن أنظمة لمعالجة التواريخ، تطبيع النصوص، معالجة القيم المفقودة، وتجميع دلالي للنصوص.

الغرض من المشروع:
- توفير خطوط أنابيب تنظيف موثوقة وقابلة للتخصيص.
- تقديم وظائف جاهزة للاستخدام في أنظمة ETL أوكنماذج ML.

## 2. الميزات الرئيسية
- تنظيف وتحويل التواريخ بصيغ متعددة إلى ISO.
- تطبيع النصوص: إزالة ضوضاء النص، تصحيح الترميز، وتطبيع الأحرف.
- استراتيجيات للتعامل مع القيم المفقودة (imputation) قابلة للتبديل.
- تجميع دلالي للنصوص لتجميع سجلات متشابهة.
- أدوات مساعدة للتشغيل كـ CLI أو واجهة Streamlit (إن وُجدت في `app.py`).

## 3. المتطلبات والبيئة
- Python 3.10+ موصى به.
- بيئة افتراضية معزولة (venv أو conda).

التثبيت الأساسي:

```bash
python -m venv .venv
# PowerShell
.venv\Scripts\Activate.ps1
# ثم:
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
````

ملاحظة: تحقق من `requirements.txt` لتعديل أو إضافة مكتبات متطلبات خاصة بمجموعات بيانات أو نماذج إضافية.

## 4. كيفية التشغيل

تشغيل سريع لتجربة تنظيف ملف CSV:

```bash
python app.py --input temp_test_upload.csv --output cleaned.csv
```

إن كان `app.py` تطبيق Streamlit (واجهة تفاعلية):

```bash
streamlit run app.py
```

مخرجات شائعة: ملف CSV نظيف، تقرير تدقيق للجودة، أو واجهة عرض النتائج.

## 5. بنية المشروع (مفصّل)

- `app.py` — نقطة الدخول للتطبيق (واجهة أو سكربت تشغيل). راجع هذا الملف لمعرفة واجهات CLI أو إعدادات Streamlit.
- `requirements.txt` — الحزم المطلوبة.
- `config/`
  - `config_presets.py` — إعدادات وتكوينات افتراضية قابلة للتعديل.
- `core/` — مكتبة الوظائف الأساسية (الكود النشط والموصى به للاستخدام):
  - `date_cleaner.py` — كشف صيغ التواريخ، تحويلها، ومعالجة القيم الشاذة.
  - `text_processor.py` — تنظيف النصوص، تطبيع، وإعداد لخطوات الـ NLP.
  - `semantic_cluster.py` — خوارزميات لتجميع النصوص دلالياً.
- `legacy/` — أمثلة سابقة، محركات قديمة، ومستندات مساعدة. استخدمها كمرجع تاريخي.
- `tests/` — اختبارات الوحدة والتكامل: شغّلها عبر `pytest`.

## 6. إعداد وتخصيص

- لتعديل إعدادات السلوك (مثل طرق Imputation أو نماذج التشابه): افتح `config/config_presets.py` وعدّل القيم أو أضف إعدادات جديدة.
- لإضافة طريقة معالجة جديدة: أنشئ ملفًا داخل `core/` واتّبع نمط الدوال الحالية، ثم أضف اختبارًا في `tests/`.

## 7. أمثلة استخدام متقدمة

- تنظيف عمود تاريخ مع قيم غير قياسية:

```python
from core.date_cleaner import normalize_dates
df['date_clean'] = normalize_dates(df['raw_date'])
```

- تشغيل تجميع دلالي على عمود نصي:

```python
from core.semantic_cluster import cluster_texts
clusters = cluster_texts(df['text_column'], threshold=0.75)
```
